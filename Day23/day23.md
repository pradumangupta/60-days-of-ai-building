# Project

## 🚀 Customer & MVP Blueprint – AI Job Search Copilot

Today, I used Claude as a Product Manager and Customer Research Assistant to convert my startup idea into a buildable MVP roadmap.

The goal was to move beyond the idea stage and define:

- Who the ideal customer is
- What problem they face
- What features should be built first
- What should not be built initially
- How to validate product-market fit

The startup idea analyzed was:

**AI Job Search Copilot** — an AI-powered career assistant designed for early-career Software, Data, and AI professionals. :contentReference[oaicite:0]{index=0}

# Customer Discovery

## 🎯 Ideal Customer Profile (ICP)

The analysis identified the primary customer as:

- Final-year students
- Early-career professionals (0–3 years)
- Career switchers entering AI/Data domains

Target Geography:

- Phase 1: India
- Phase 2: US, Canada, UK, Australia

Behavioral Traits:

- Active on LinkedIn and Naukri
- Uses ChatGPT regularly
- Applies to 10+ jobs per week while job hunting. :contentReference[oaicite:1]{index=1}

# Buyer Persona

## 👨‍💻 "Switching Sahil"

A 24–27-year-old professional transitioning into AI/Data roles.

### Goals

- Land an AI/Data job within 6–12 months
- Increase interview callback rates

### Frustrations

- Resume tailoring is exhausting
- ATS optimization is unclear
- Difficult to track applications

Success Metric:

> More interview callbacks per application. :contentReference[oaicite:2]{index=2}

# Customer Journey

The customer journey was mapped across four stages:

## Awareness

Users discover the tool through:

- LinkedIn posts
- Build-in-public content
- Peer recommendations

## Consideration

Users compare the solution against:

- ChatGPT
- Naukri
- Careerflow

## Purchase

Users convert after experiencing immediate value such as:

- ATS Match Score
- Tailored Resume Draft

## Retention

Users return for:

- Every new application
- Resume optimization
- Application tracking
- Interview preparation. :contentReference[oaicite:4]{index=4}

# MVP Recommendation

## ✅ Build First

Claude recommended building only the core value wedge:

1. Resume + JD Upload
2. ATS Match Score
3. AI Resume Rewrite Suggestions
4. AI Cover Letter Generation
5. Lightweight Application Tracker

These features deliver the strongest "Aha Moment" for users. :contentReference[oaicite:5]{index=5}

---

## ❌ Do Not Build Yet

Features intentionally excluded from V1:

- Full Job Aggregator
- Browser Extension
- AI Mock Interview Simulator
- Native Mobile App
- Recruiter SaaS Platform

The recommendation was to validate the core workflow before expanding. :contentReference[oaicite:6]{index=6}

# Key Learnings

## 1. Start Narrow

Successful products solve one critical problem exceptionally well.

## 2. Customer Clarity Is Essential

Understanding customer pain points determines product direction.

## 3. MVP ≠ Full Product

An MVP should validate assumptions, not implement every idea.

## 4. Product-Market Fit Comes Before Scaling

Validation should happen before investing heavily in features.

---

# Biggest Insight

The fastest path to product-market fit is:

✅ Identify one painful problem

✅ Define one clear customer segment

✅ Build one strong "Aha Moment"

✅ Validate willingness to pay

Everything else can come later.
